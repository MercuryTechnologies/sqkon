# Contributing

Sqkon is maintained by [Mercury](https://mercury.com) and welcomes external contributions — bug reports, documentation fixes, and pull requests.

The canonical contributor guide lives at the repository root: **[CONTRIBUTING.md](https://github.com/MercuryTechnologies/sqkon/blob/main/CONTRIBUTING.md)**. It covers the commit format, branch conventions, CI expectations, and project rules. Please read it before opening your first PR.

## Local development

Sqkon uses Gradle and the [Just the Docs](https://just-the-docs.com) Jekyll theme. Java 21 and Ruby 3+ are required.

### Library

```bash
# Run the JVM test suite (primary dev loop)
./gradlew jvmTest

# Run a single test class
./gradlew jvmTest --tests "*.KeyValueStorageTest"

# Verify SQLDelight schema migrations
./gradlew verifySqlDelightMigration

# Publish a snapshot to your local Maven for downstream testing
./gradlew publishToMavenLocal
```

### Documentation site

```bash
cd docs
bundle install
bundle exec jekyll serve --livereload
# → http://127.0.0.1:4000/sqkon/
```

The site rebuilds on save; the [API reference]({{ '/api/' | relative_url }}) is a separate Dokka build and is not regenerated by `jekyll serve`.
